## **AI Basic**
| Concept | Acronym | Definition |
| --- |---|---|
| **LLM**	 |**Large Language Model** | A model trained on massive text datasets, capable of generating language, understanding context, and performing a wide range of text based tasks.|
| **MCP** 	| **Model Context Protocol**	| **A standard way for LLMs to use tools.**<br/>   •	MCP defines a uniform interface that connects an LLM with external systems such as databases, file systems, GitHub, Slack, or internal APIs.<br/> •	It does not decide what the model should do — it only standardizes how tools are exposed and how the model can call them.<br/> •	An MCP server is the component that provides these tools to the model. |
| **MCP server** | |  **An MCP server is the component that exposes tools and data sources to an LLM using the Model Context Protocol**.<br/> •	Provides tools: It defines and serves tools such as database queries, file access, APIs, or integrations (e.g., GitHub, Slack).<br/> •	Speaks MCP: It follows the MCP specification so that any compatible LLM can discover, list, and call its tools in a standard way.<br/> •	Stateless about logic: It doesn’t decide when or why tools are used — it only makes them available. The agent or LLM logic decides what to call. |
| **RAG**	 |**Retrieval-Augmented Generation** | **LLMs augmented/ enhanced with retrieved knowledge**<br/> It is about what the model knows at runtime<br/>The model itself stays frozen no retraining or fine tuning. <br/>When a user asks a question, a retriever fetches relevant documents (PDFs, code, vector databases, etc.) and injects them into the prompt.<br/>RAG is ideal for:<br/> - internal knowledge bases,<br/> - fresh or private data,<br/> - reducing hallucinations.| Limitation: RAG does not take actions. It only improves the model’s answers.|
|**AI Agents** | | **LLMs that can take actions and make decisions.** <br/> An agent operates in a loop: observe → reason → decide → act → repeat.<br/> It can:<br/> •	call tools,<br/>•	write and run code,<br/>•	browse the internet,<br/>•	store memory,<br/> •	delegate tasks, <br/>•	operate with varying levels of autonomy.|
